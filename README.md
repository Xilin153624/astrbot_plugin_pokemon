# 精灵对战插件 for AstrBot

一款类宝可梦的回合制精灵对战插件，支持属性克制、捕捉养成、玩家对战，测试模式下战斗后自动恢复状态，开箱即用。

## ✨ 功能特点
- **9种属性**：火、水、草、飞行、电、地面、机械、暗、光，完善的属性克制矩阵。
- **三个初始精灵家族**（各三阶进化）：
  - 水甲龟 → 沼甲龟 → 海玄龟（水 / 水+地面）
  - 赤火猪 → 炎火猪 → 炽火野猪（火 / 火+地面）
  - 仙人球 → 仙人掌 → 巨型仙人掌（草 / 草+地面）
- **精灵培养**：捕捉、放生、设置首发、查看队伍。
- **玩家对战**：通过QQ号发起挑战，回合制同时出招，精灵倒下后必须手动换人。
- **测试福利**：战斗结束后所有精灵的HP和PP自动恢复至战斗前状态。
- **数据持久化**：玩家数据、精灵图鉴均存储在 `data/pokemon/` 目录下。

## 📦 安装方法
1. 将本插件文件夹 `astrbot_plugin_pokemon` 放置于 AstrBot 的 `data/plugins/` 目录下。
2. 重启 AstrBot 或重载插件。
3. 首次运行会自动生成 `data/pokemon/pokedex.json`（精灵图鉴）和 `data/pokemon/trainers.json`（玩家数据）。

## 📋 命令列表
| 命令 | 说明 |
|------|------|
| `我的精灵` | 查看自己的精灵队伍，当前出战精灵前有 ★ 标记 |
| `选择初始 1/2/3` | **仅首次有效**：1=水甲龟(水), 2=赤火猪(火), 3=仙人球(草) |
| `捕捉` | 随机获得一只初始精灵（水/火/草），等级1，可重复使用 |
| `选择 N` | **非战斗时**：设置第N只精灵为首发（★）<br>**战斗中**：临时切换到第N只精灵出战（需精灵未倒下） |
| `技能 N` | 战斗中使用当前精灵的第N个技能（N=1~4） |
| `挑战 QQ号` | 向指定玩家发起对战请求 |
| `接受 QQ号` | 接受某玩家的挑战请求 |
| `拒绝 QQ号` | 拒绝某玩家的挑战请求 |
| `换人 N` | 战斗中手动切换到第N只精灵（需精灵未倒下） |
| `退出战斗` | 强制结束当前对战，双方精灵状态恢复至战斗前 |
| `放生 N` | 永久释放队伍中的第N只精灵（不可在战斗中使用，不可放生当前出战精灵） |
| `图鉴` | 查看所有可获得的精灵列表及编号 |
| `帮助` | 显示本帮助信息 |

## ⚔️ 战斗规则
- **回合制**：每回合双方同时选择技能（`技能 N`），同时计算伤害。
- **精灵倒下**：若精灵HP归零，该精灵倒下，必须手动换人才能继续战斗（使用 `选择 N` 或 `换人 N`）。
- **战斗结束**：一方所有精灵倒下则战斗结束，另一方获胜；若双方同时无精灵，则为平局。
- **状态恢复**：战斗结束后（无论胜负或平局），所有精灵的HP和PP自动恢复至战斗前状态（测试模式福利）。
- **首发机制**：非战斗时使用 `选择 N` 设置首发精灵（★标记）；战斗开始时会自动派出首发精灵；战斗中换人不会改变首发记录。

## 💾 数据文件
插件运行后会在 `AstrBot/data/pokemon/` 下生成两个文件：
- `pokedex.json`：精灵图鉴，包含所有精灵的基础数据（种族值、技能等），可自行编辑扩展。
- `trainers.json`：玩家数据，包括拥有的精灵、队伍、首发索引等。


## 🛠️ 自定义扩展
- 如需添加新精灵，请编辑 `pokedex.json`，按现有格式添加条目。
- 如需调整属性克制，修改 `config.py` 中的 `TYPE_CHART` 字典。
- 技能效果目前仅实现伤害技能，变化技能（如提升防御）暂未实现，可自行在 `battle.py` 中扩展。

## 📝 注意事项
- 命令中的 `N` 均为精灵在队伍中的序号（从1开始）。
- 战斗中无法使用 `选择初始`、`捕捉`、`放生` 等非战斗指令。
- 如果遇到战斗卡死或异常，可使用 `退出战斗` 强制结束并恢复状态。
- 首次使用请先 `选择初始` 获得第一只精灵，或直接用 `捕捉` 获取。
